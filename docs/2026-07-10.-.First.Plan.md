# GeoNotes Implementation Plan

## Context

Greenfield build in an empty directory (`/home/victor/dev/apps/GeoNotesClaude`, not yet git-initialized). GeoNotes is an offline-first PWA for short location-pinned notes, architected for future Capacitor packaging (iOS/Android) from the same codebase. Backend on Cloudflare Pages + Pages Functions + D1. The user will activate the Cloudflare infrastructure; we build everything so it runs locally via `wrangler pages dev` today and deploys unchanged.

## Decisions (confirmed with user)

- Stack: React + Vite + TypeScript, vite-plugin-pwa (Workbox), Dexie (IndexedDB) for local store + sync outbox.
- Email OTP delivery: stubbed behind an `EmailSender` interface (dev mode logs/returns the code). Passkeys fully implemented. Real provider wired later.
- GPS thresholds: unlock "+" instantly at accuracy <= 10m. Fallback: once accuracy is within 20 to 50m, start a 5s timer; if 10m isn't reached in 5s, unlock with the current fix. GPS polling is killed the moment the fix is accepted.
- Nearby radius: a note counts as "at your location" within `max(25m, fix accuracy)`.
- Reverse geocoding: Nominatim (OSM), proxied through a Pages Function with proper User-Agent, edge caching, and "© OpenStreetMap contributors" attribution in the UI.

## Data model (D1)

- `users(id, email UNIQUE, created_at)`
- `credentials(id, user_id, public_key, counter, transports, created_at)` for WebAuthn
- `email_codes(email, code_hash, expires_at, attempts)` for OTP
- `sessions(token_hash, user_id, expires_at)` bearer tokens (not cookies, for Capacitor compatibility)
- `notes(id TEXT PK, user_id, text CHECK(length(text)<=512), lat REAL, lng REAL, address TEXT NULL, created_at, updated_at)`

Note ids are client-generated UUIDs so offline creation needs no server round trip. Location is immutable after creation (enforced server-side: upsert ignores lat/lng on existing rows). Deletes are hard `DELETE` statements, no tombstones; conflict resolution is last-write-wins on `updated_at`.

## Key behaviors

**Geolocation lifecycle (useGeolocation):** on main screen mount, `watchPosition({enableHighAccuracy: true})`. Live accuracy shown in meters in an AccuracyBadge. Lock rules as in Decisions above; on lock, `clearWatch` immediately (battery). Re-arms only when returning to the main screen. Tapping "+" snapshots the locked coords into the editor; moving afterward never changes them.

**Main screen:** instantly lists all cached notes from Dexie (sorted by updated_at) while GPS acquires, then progressively filters/sorts to nearby-first as fixes arrive. Notes within the nearby radius are highlighted as "here". Bottom-right floating "+" button, disabled with a subtle progress state until the fix locks. Link/toggle to the All Notes screen (full list with addresses and distances).

**Editor:** textarea capped at 512 chars with live `n / 512` counter, `**bold**` preview, shows the locked address (reverse-geocoded when online, backfilled on sync when offline). Editing an existing note reuses the screen with location read-only. Delete asks for confirmation then hard-deletes locally and enqueues the server delete.

**Sync engine (lib/sync.ts):** outbox table records `{op: upsert|delete, noteId, payload, queued_at}`. Flush triggers: app start, `online` event, after any local mutation while online. Flush pushes outbox in order, then pulls the full note list (payloads are tiny; this satisfies "all notes offline") and replaces the local cache except rows with pending outbox entries. Notes created offline without an address get reverse-geocoded during flush and re-upserted.

**Auth:** @simplewebauthn/browser + @simplewebauthn/server (v10+, WebCrypto-based, runs on Workers). Flows: enter email -> passkey login if credentials exist, else email code -> verify -> offer passkey enrollment. OTP codes are 6 digits, hashed in D1, 10 min expiry, max 5 attempts. `EmailSender` interface with a `DevEmailSender` that logs the code (and returns it in the response when `ENVIRONMENT=dev`). Session bearer token stored client-side; app works fully offline against cached session + Dexie data, auth only re-checked when syncing.

**i18n:** en/es/pt JSON dictionaries, auto-detected, all UI strings externalized from day one.

**PWA:** vite-plugin-pwa precaches the app shell for full offline boot; `/api/*` is network-only (sync is handled by the outbox, not Workbox background sync). Manifest with maskable icons, theme colors for both schemes.

## Implementation order

1. **Scaffold:** git init, Vite React TS app, vite-plugin-pwa, wrangler.toml with D1 binding + local dev, migrations/0001_init.sql, theming tokens, i18n skeleton with the three locales, placeholder icon set.
2. **Local-first core (no backend):** Dexie schema, useGeolocation with the lock/fallback logic, MainScreen with accuracy badge + nearby filtering, Editor with counter/bold/hard delete, AllNotesScreen. App is fully usable offline at this point.
3. **Backend:** notes API (list/upsert/delete) on Pages Functions + D1, geocode proxy with caching and attribution.
4. **Auth:** passkey register/login, stubbed email OTP, sessions, AuthScreen.
5. **Sync:** outbox flush, pull reconciliation, address backfill, online/offline indicator.
6. **Polish:** real icon set, manifest metadata, empty states, verification pass.
